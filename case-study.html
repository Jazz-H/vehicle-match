01

The problem

Car shoppers face choice overload. A modern lineup spans dozens of models across body types, powertrains and price bands — and the tools meant to help (spec sheets, filter-heavy search, comparison tables) all assume the shopper already knows what they want.

Most don't. They know how they live: a growing family, a long commute, weekend towing, a tight budget. For the dealer, that mismatch is expensive — undirected browsers bounce, and the ones who do reach out arrive cold, so every conversation starts from zero.

02

The approach

Instead of another filter grid, Vehicle Match runs a guided, three-question conversation and returns a single confident recommendation.

1

Type

What body style are you shopping for — or keep it open and let priorities decide.

2

Priorities

Pick up to three things that matter most, in order. The first pick counts the most.

3

Preferences

A few hard limits — budget, seating, drivetrain — so only what actually fits is matched.

03

The solution

Transparent engine

An explainable match, not a black box

Match % is a weighted average of the shopper's chosen priority scores (first pick heaviest, 3-2-1) after hard/soft filters. Pure, deterministic and unit-tested — auditable, not hand-wavy. When filters are too strict, they relax gracefully and the result says so.

Converts at peak intent

A result that captures the lead

The match screen ends in a personalized-quote form. Interest is captured the instant the shopper sees "this is the one" — not with a generic contact form bolted on elsewhere.

04

The data-driven insight

Every step and every result is logged as an anonymous event, so the shopper's journey becomes the dealer's insight. The same interactions feed a live dealer dashboard: unique sessions, captured leads, conversion rate, and the most-wanted body type — plus a priority-popularity chart and a session funnel.

That turns a lead-gen widget into a demand-sensing tool. A dealer can see, this week, which priorities lead, which body types win, and where shoppers drop off — a concrete signal for how to stock, price and follow up.

05

How it's built

A deliberately lean, dependency-free stack — proof that a real business tool doesn't need a heavy framework.

Front endVanilla HTML/CSS/JS — no framework, no build step. Static and instant.
DesignThe Motorpool system — Saira Condensed + IBM Plex, a steel & redline palette.
EngineA pure scoring module with Node unit tests — the logic is auditable.
Data planeSupabase (Postgres). The browser writes leads + events directly; row-level security keeps the public key insert-only, and dashboard reads require an authenticated dealer session.
HostingGitHub Pages — static, zero server to run.
CraftFocus management, aria-live announcements, keyboard nav, ≥44px targets, reduced-motion, and full responsiveness.
